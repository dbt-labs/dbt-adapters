## dbt-snowflake 1.12.0-rc1 - July 09, 2026

### Features

- snowflake__generate_database_name reads catalog_database from the catalog relation via build_catalog_relation; when set it takes highest priority over model database config and target.database. catalog_database is now surfaced on BuiltInCatalogRelation and IcebergRestCatalogRelation. ([#2004](https://github.com/dbt-labs/dbt-adapters/issues/2004))

### Fixes

- Allow Workload Identity Federation (e.g. OIDC) auth to use a token without a user by exempting the workload_identity authenticator from the token/user profile validation ([#](https://github.com/dbt-labs/dbt-adapters/issues/))

### Dependencies

- bump dbt-adapters lower bound to 1.24.4 to pick up catalog_database support ([#2053](https://github.com/dbt-labs/dbt-adapters/pull/2053))



## dbt-snowflake 1.12.0-b2 - June 15, 2026

### Fixes

- Use SHOW USER FUNCTIONS in list_function_relations_without_caching to filter built-in functions server-side; eliminates per-schema large-result chunked S3 fetches that saturated customer-owned VPCe stage endpoints ([#1973](https://github.com/dbt-labs/dbt-adapters/issues/1973))

### Dependencies

## dbt-snowflake 1.12.0-b1 - May 22, 2026

### Breaking Changes

- Add transient support for Snowflake dynamic tables via `transient` config option and `snowflake_default_transient_dynamic_tables` behavior flag. ([#716](https://github.com/dbt-labs/dbt-adapters/issues/716))

### Features

- Add COPY GRANTS support for dynamic tables to inherit object-level privileges during table creation and replacement ([#1154](https://github.com/dbt-labs/dbt-adapters/issues/1154))
- Introduction of Workload Identity Federation as a supported method of authentication to Snowflake ([#1234](https://github.com/dbt-labs/dbt-adapters/issues/1234))
- Add information on number of rows in CTAS ([#1531](https://github.com/dbt-labs/dbt-adapters/issues/1531))
- Add snowflake_initialization_warehouse parameter for dynamic tables ([#1566](https://github.com/dbt-labs/dbt-adapters/issues/1566))
- Add immutable_where parameter for dynamic tables ([#none](https://github.com/dbt-labs/dbt-adapters/issues/none))
- Add support for Snowflake Programmatic Access Token (PAT) authentication ([#1688](https://github.com/dbt-labs/dbt-adapters/issues/1688))
- Support PyPI packages for Python UDFs - when packages are specified in function config, they are templated into the CREATE statement (PACKAGES = (...)) for scalar and aggregate Python UDFs ([#1737](https://github.com/dbt-labs/dbt-adapters/issues/1737))
- Add scheduler configuration support for Snowflake dynamic tables, enabling dbt-managed refresh scheduling that is decoupled from upstream and downstream dynamic table dependencies. ([#none](https://github.com/dbt-labs/dbt-adapters/issues/none))
- Add change_tracking support for regular Snowflake tables. ([#722](https://github.com/dbt-labs/dbt-adapters/issues/722))
- Add refresh_warehouse config for Snowflake dynamic tables, allowing a separate warehouse for self-refresh independent of the DDL execution warehouse ([#1193](https://github.com/dbt-labs/dbt-adapters/issues/1193))
- Add support for `iceberg_version` config to enable Iceberg V3 table creation via model config or `catalogs.yml` `adapter_properties`. ([#1779](https://github.com/dbt-labs/dbt-adapters/issues/1779))
- Add support for the Snowflake COPY TAGS parameter in CREATE OR REPLACE TABLE, clone operations, and built-in Iceberg table creation ([#1857](https://github.com/dbt-labs/dbt-adapters/issues/1857))
- Added support for JS UDFs ([#1859](https://github.com/dbt-labs/dbt-adapters/issues/1859))
- Added support for --empty in dbt seed ([#1865](https://github.com/dbt-labs/dbt-adapters/issues/1865))
- Add `tmp_relation_type=transient` support for incremental models, enabling Snowflake lineage tracking; add `snowflake__resolve_incremental_tmp_relation` dispatch macro for controlling tmp relation destination ([#1893](https://github.com/dbt-labs/dbt-adapters/issues/1893))
- Add bridge_v2_catalog support: _v2_to_v1_type, _v2_table_format (uppercase), _translate_v2_properties (linked field map), and Capability.CatalogsV2 ([#1920](https://github.com/dbt-labs/dbt-adapters/issues/1920))

### Fixes

- Use INSERT OVERWRITE to make dbt seed atomic on Snowflake, preventing concurrent readers from seeing an empty table between truncate and insert ([#740](https://github.com/dbt-labs/dbt-adapters/issues/740))
