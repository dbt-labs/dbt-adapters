## dbt-snowflake 1.11.6 - June 24, 2026

### Fixes

- When persist_docs.columns is enabled, dbt now warns about columns defined in YAML that don't exist in the database and filters them via the validate_doc_columns helper. ([#1690](https://github.com/dbt-labs/dbt-adapters/issues/1690))
- Fix Python incremental `__dbt_tmp` being created as a permanent table, restore the `transient` config for Python `materialized='table'`, and accept `tmp_relation_type='table'` for Python models. ([#1928](https://github.com/dbt-labs/dbt-adapters/issues/1928))
- Add snowflake__persist_docs override and warn from snowflake__alter_column_comment so persist_docs emits the missing-column warning on Snowflake without an extra get_columns_in_relation call ([#1937](https://github.com/dbt-labs/dbt-adapters/issues/1937))
