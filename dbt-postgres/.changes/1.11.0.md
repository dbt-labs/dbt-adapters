## dbt-postgres 1.11.0 - July 16, 2026

### Features

- Add autocommit support to eliminate idle-in-transaction states ([#1608](https://github.com/dbt-labs/dbt-adapters/issues/1608))
- Added support for --empty in dbt seed ([#1865](https://github.com/dbt-labs/dbt-adapters/issues/1865))
- Add support for python 3.14 ([#12098](https://github.com/dbt-labs/dbt-adapters/issues/12098))

### Fixes

- Enable discovering pre-existing functions by updating `list_relations_without_caching` to include function relations ([#1488](https://github.com/dbt-labs/dbt-adapters/issues/1488))
- equals macro now uses IS NOT DISTINCT FROM clause for comparison ([#1663](https://github.com/dbt-labs/dbt-adapters/issues/1663))
- When persist_docs.columns is enabled, dbt now warns about columns defined in YAML that don't exist in the database and eliminates a duplicate get_columns_in_relation roundtrip via the validate_doc_columns helper. ([#1690](https://github.com/dbt-labs/dbt-adapters/issues/1690))

### Dependencies

- Bump dbt-adapters floor to 1.24.1 for catalogs v2 and JS UDF support ([#1965](https://github.com/dbt-labs/dbt-adapters/pull/1965))
- Add upper bound `<2.0` on `dbt-core` to prevent pip from resolving to dbt-core 2.0 (Fusion engine), which is incompatible with the python adapter ([#2013](https://github.com/dbt-labs/dbt-adapters/pull/2013))
