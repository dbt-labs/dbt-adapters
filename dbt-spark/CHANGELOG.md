## dbt-spark 1.11.0-b1 - May 22, 2026

### Features

- Added support for --empty in dbt seed ([#1865](https://github.com/dbt-labs/dbt-adapters/issues/1865))

### Fixes

- When persist_docs.columns is enabled, dbt now warns about columns defined in YAML that don't exist in the database and eliminates a duplicate get_columns_in_relation roundtrip via the validate_doc_columns helper. Also fixed hatch.toml to use local dbt-adapters install. ([#1690](https://github.com/dbt-labs/dbt-adapters/issues/1690))
- Stop silently swallowing errors in list_relations_without_caching ([#1289](https://github.com/dbt-labs/dbt-adapters/issues/1289))
- Fix use_ssl=True SASL incompatibility by importing PureSASLClient from pyhive.sasl_compat instead of puresasl.client ([#1846](https://github.com/dbt-labs/dbt-adapters/issues/1846))
- Fix empty catalog.json when using Iceberg v2 tables with dbt docs generate ([#485](https://github.com/dbt-labs/dbt-adapters/issues/485))

### Dependencies

- Add support for Spark v4.x ([#1537](https://github.com/dbt-labs/dbt-adapters/pull/1537))
- Update thrift to use at least 0.20.0 ([##1727](https://github.com/dbt-labs/dbt-adapters/pull/#1727))
- Bump dbt-adapters floor to 1.24.1 for catalogs v2 and JS UDF support ([#1968](https://github.com/dbt-labs/dbt-adapters/pull/1968))
