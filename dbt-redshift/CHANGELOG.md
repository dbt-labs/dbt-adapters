## dbt-redshift 1.11.0-rc1 - April 06, 2026

### Features

- Add support for granting privileges to Redshift groups and roles via grants config using 'group:' and 'role:' prefixes ([#647](https://github.com/dbt-labs/dbt-adapters/issues/647))
- Redshift: Use native ALTER COLUMN TYPE for VARCHAR and VARBYTE column size changes; fall back to default for other type changes ([#649](https://github.com/dbt-labs/dbt-adapters/issues/649))
- Adds support for Redshift session `query_group` so dbt runs can be tagged for WLM and query logging ([#648](https://github.com/dbt-labs/dbt-adapters/issues/648))
- Added behavior flag `redshift_use_show_apis` (default: false) for opting into SVV_*-based metadata queries, required for Datasharing/cross-database. ([#1621](https://github.com/dbt-labs/dbt-adapters/issues/1621))
- Use SVV system views for relation listing when `redshift_use_show_apis` is enabled ([#1622](https://github.com/dbt-labs/dbt-adapters/issues/1622))
- Migrate `list_schemas` and `check_schema_exists` to SVV_ALL_SCHEMAS when `redshift_use_show_apis` flag is enabled, enabling cross-database schema listing ([#1623](https://github.com/dbt-labs/dbt-adapters/issues/1623))
- Migrate `get_columns_in_relation` to use `svv_all_columns`, enabling cross-database column introspection for Datasharing ([#1624](https://github.com/dbt-labs/dbt-adapters/issues/1624))
- list_relations - using SHOW TABLES command to get the relation details instead of using svv_* based tables ([#1669](https://github.com/dbt-labs/dbt-adapters/issues/1669))
- Migrate get_columns_in_relation to use SHOW COLUMNS instead of svv_all_columns ([#1672](https://github.com/dbt-labs/dbt-adapters/issues/1672))
- Migrate `list_schemas` and `check_schema_exists` to use `SHOW SCHEMAS` ([#1674](https://github.com/dbt-labs/dbt-adapters/issues/1674))
- Migrate get_show_grant_sql to use SHOW GRANTS ON TABLE ([#1677](https://github.com/dbt-labs/dbt-adapters/issues/1677))
- Support cross-database `create_schema` and `drop_schema` when `redshift_use_show_apis` is enabled, and deprecate `ra3_node` as an alias for the behavior flag. ([#1693](https://github.com/dbt-labs/dbt-adapters/issues/1693))
- Migrate catalog macros (`get_catalog`, `get_catalog_relations`) to use SHOW TABLES and SVV_REDSHIFT_COLUMNS when `redshift_use_show_apis` is enabled ([#1625](https://github.com/dbt-labs/dbt-adapters/issues/1625))
- Support metadata-based freshness across databases using SHOW TABLES FROM SCHEMA when redshift_use_show_apis is enabled ([#1626](https://github.com/dbt-labs/dbt-adapters/issues/1626))
- Add support for per-model database switching using USE <database> before model execution and RESET USE after, enabling models targeting a non-default db ([#1786](https://github.com/dbt-labs/dbt-adapters/issues/1786))
- Add `allow_concurrent_drops` credential option to skip the DROP RLock for projects with no downstream views ([#1771](https://github.com/dbt-labs/dbt-adapters/issues/1771))

### Fixes

- Adding friendlier error message when dist is not string for redshift ([#651](https://github.com/dbt-labs/dbt-adapters/issues/651))
- Fix dbt clone failing when persist_docs adds column comments to late binding views ([#622](https://github.com/dbt-labs/dbt-adapters/issues/622))
- When persist_docs.columns is enabled, dbt now warns about columns defined in YAML that don't exist in the database and eliminates a duplicate get_columns_in_relation roundtrip via the validate_doc_columns helper. ([#1690](https://github.com/dbt-labs/dbt-adapters/issues/1690))
- Handle missing table_subtype and decouple ra3_node from use_show_apis ([#1745](https://github.com/dbt-labs/dbt-adapters/issues/1745))
- Use the ra3_node config where it was used earlier. ([#1750](https://github.com/dbt-labs/dbt-adapters/issues/1750))
- Remove ra3_node deprecation log until SHOW APIs are GA ([#1773](https://github.com/dbt-labs/dbt-adapters/issues/1773))
- Add `datasharing` profile credential as an alternative to the `redshift_use_show_apis` behavior flag for enabling SHOW/SVV_* APIs; either can be used to enable cross-database Datasharing support. ([#1621](https://github.com/dbt-labs/dbt-adapters/issues/1621))
- Add cross-database support for create_schema and drop_schema by issuing USE <database> / RESET USE around schema operations targeting a non-default database ([#1786](https://github.com/dbt-labs/dbt-adapters/issues/1786))
- Fix grant reconciliation failure on clusters with datashares by filtering reserved Redshift roles (ds:*, sys:*) from standardize_grants_dict ([#1808](https://github.com/dbt-labs/dbt-adapters/issues/1808))
- Filter out internal `pg.dropped` phantom columns from `SHOW COLUMNS` results to prevent schema-change errors during cross-database column type alterations ([#1816](https://github.com/dbt-labs/dbt-adapters/issues/1816))
- Revert breaking grants behavior from 1597 behind opt-in flag `redshift_grants_extended` to prevent unintended revocation of externally-managed role and group grants ([#647](https://github.com/dbt-labs/dbt-adapters/issues/647))

### Under the Hood

- Default `redshift_skip_autocommit_transaction_statements` to `False` to preserve legacy transaction behavior. Users can opt-in to skip unnecessary transaction statements by setting the flag to `True`. ([#1538](https://github.com/dbt-labs/dbt-adapters/issues/1538))
- Fix bug where `retryable_exceptions` is not iterable with `retry_all` config param set ([#1587](https://github.com/dbt-labs/dbt-adapters/issues/1587))
- Add datasharing profile variant tests for incremental on_schema_change, incremental unique_key, materialized view basic, and UDFs to catch metadata regressions when SHOW APIs are enabled ([#1808](https://github.com/dbt-labs/dbt-adapters/issues/1808))
