## dbt-redshift 1.11.0-rc3 - May 22, 2026

### Breaking Changes

- Remove the `redshift_use_show_apis` behavior flag. Use the `datasharing` profile credential instead to enable SHOW/SVV_* APIs for cross-database Datasharing support. ([#1621](https://github.com/dbt-labs/dbt-adapters/issues/1621))

### Features

- Add `drop_without_cascade` profile credential to omit CASCADE from DROP TABLE/VIEW/MATERIALIZED VIEW statements ([#1772](https://github.com/dbt-labs/dbt-adapters/issues/1772))

### Fixes

- Enable discovering pre-existing functions by updating `list_relations_without_caching` to include function relations ([#1488](https://github.com/dbt-labs/dbt-adapters/issues/1488))
- Fix 'Tried to commit transaction but it does not have one open' crash when a transient Redshift error triggers a retry during DROP in fresh_transaction ([#1698](https://github.com/dbt-labs/dbt-adapters/issues/1698))
- Auto-retry 'conflict with concurrent transaction' errors during DROP statements in Redshift ([#626](https://github.com/dbt-labs/dbt-adapters/issues/626))
- Quote database, schema, and table identifiers in SHOW TABLES/SHOW COLUMNS/SHOW GRANTS/SHOW SCHEMAS so hyphenated names are accepted by Redshift's SHOW APIs ([#1884](https://github.com/dbt-labs/dbt-adapters/issues/1884))

### Dependencies

- Bump dbt-adapters floor to 1.24.1 for catalogs v2 and JS UDF support ([#1966](https://github.com/dbt-labs/dbt-adapters/pull/1966))
